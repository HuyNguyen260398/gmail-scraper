# Codebase Structure

## 1) Top-Level Map

| Path | Purpose | Evidence |
|------|---------|----------|
| `src/` | Application source for auth, Gmail API access, and CLI entry | `README.md`, `src/main.py`, `src/auth.py`, `src/gmail_client.py` |
| `config/` | Runtime OAuth files expected/generated by the app | `README.md`, `src/auth.py`, `.gitignore` |
| `tests/` | Pytest coverage for exporter behavior, Gmail body parsing, and CLI export wiring | `AGENTS.md`, `tests/` |
| `requirements.txt` | Python dependency manifest | `requirements.txt` |
| `.env.example` | Example CLI default configuration | `.env.example`, `src/main.py` |
| `.gitignore` | Excludes secrets, virtualenvs, Python cache files, and local env files | `.gitignore` |
| `README.md` | Setup, usage, Gmail query examples, and deployment notes | `README.md` |
| `CLAUDE.md` | Maintainer guidance for commands, architecture, and editing constraints | `CLAUDE.md` |

## 2) Entry Points

- Main runtime entry: `src/main.py`.
- Secondary entry points: none found.
- How entry is selected: commands in `README.md` and `CLAUDE.md` run `python3 src/main.py` or `uv run python3 src/main.py`, which puts `src/` on `sys.path`.

## 3) Module Boundaries

| Boundary | What belongs here | What must not be here |
|----------|-------------------|------------------------|
| `src/main.py` | CLI parsing, loading `.env`, calling the Gmail client, printing message metadata and body content | OAuth flow details or MIME parsing logic |
| `src/auth.py` | Gmail OAuth scopes, credential loading, token refresh, first-run consent flow, token file write | Gmail message listing or CLI output |
| `src/gmail_client.py` | Gmail API service construction, message listing/fetching, email dataclass, MIME body extraction | Argument parsing, environment loading, token file path policy |
| `config/` | Local OAuth credential and token files at runtime | Source code or committed secrets |

## 4) Naming and Organization Rules

- File naming pattern: Python `snake_case`, e.g. `gmail_client.py`.
- Directory organization pattern: small layer-based layout under `src/`.
- Import aliasing or path conventions: flat imports inside `src/`, e.g. `from gmail_client import GmailClient` and `from auth import get_credentials`; no package prefix or path aliases.
